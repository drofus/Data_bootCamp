Econom√≠a unitaria ‚öñÔ∏è#

La econom√≠a unitaria eval√∫a la rentabilidad por unidad: puede ser por producto, por cliente o por transacci√≥n. Este an√°lisis permite tomar decisiones estrat√©gicas como escalar campa√±as, ajustar precios o modificar el modelo de negocio.

Econom√≠a unitaria por ventas üíπ#

Mide cu√°nto ganas por cada venta, restando los costos asociados. Es √∫til para entender si vender m√°s unidades realmente significa mayor rentabilidad.

Econom√≠a unitaria por ventas: crear un modelo üíª#

Crear un modelo de econom√≠a unitaria te permite simular cambios: ¬øqu√© pasa si el costo sube? ¬øy si suben las ventas?

def modelo_venta(precio, costo, unidades):
    ingreso = precio * unidades
    costo_total = costo * unidades
    ganancia = ingreso - costo_total
    return {
        'Ingreso': ingreso,
        'Costo': costo_total,
        'Ganancia': ganancia,
        'Ganancia por unidad': precio - costo
    }

print(modelo_venta(precio=90, costo=50, unidades=1000))

{'Ingreso': 90000, 'Costo': 50000, 'Ganancia': 40000, 'Ganancia por unidad': 40}

Econom√≠a unitaria por cliente: LTV y CAC ü™™#

CAC (Costo de adquisici√≥n del cliente): cu√°nto cuesta atraer un nuevo cliente.

LTV (Valor del cliente a lo largo del tiempo): cu√°nto dinero genera un cliente promedio.

clientes = pd.DataFrame({
    'cliente_id': range(1, 6),
    'ingresos_totales': [300, 500, 200, 400, 150],
    'num_meses_activo': [3, 5, 2, 4, 1],

Sesiones de usuario üñ•Ô∏è#

Una sesi√≥n es una secuencia de eventos consecutivos sin largos per√≠odos de inactividad (ej. 30 minutos).

eventos['timestamp'] = pd.to_datetime(eventos['timestamp'])
eventos = eventos.sort_values(['usuario_id', 'timestamp'])

eventos['nueva_sesion'] = eventos['diferencia'] > pd.Timedelta(minutes=30)
